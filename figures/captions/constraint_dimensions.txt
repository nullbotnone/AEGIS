Figure 4: AEGIS Constraint Dimensions. The framework enforces five orthogonal constraint dimensions: (1) Data Access—monitors file paths and volume thresholds; (2) Network—controls endpoint allowlists and egress policies; (3) Tool Invocation—maintains allow/deny lists for agent tool usage; (4) Execution—enforces runtime limits, memory bounds, and node allocation constraints; (5) Data Flow—tracks project boundary crossings and exfiltration budgets. Each dimension is continuously monitored by the Attestation Engine and evaluated by the Policy Verifier.
