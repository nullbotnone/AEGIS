\section{Conclusion}

This paper argues that the main security problem introduced by AI agents in HPC
is not unauthorized access by an outsider, but misuse of legitimate authority by
a hijacked authorized agent. Once an agent is trusted with project data, shared
storage, tools, and network access, conventional authentication and access
control no longer answer the critical question: whether the agent's runtime
behavior still matches its authorized task.

We introduced behavioral attestation as a practical security primitive for this
setting and presented \projectname~, a system that compiles behavioral constraints,
monitors agent actions with \ebpf-based syscall interception, verifies evidence
against constraint profiles, correlates behavior across agents, and translates
verdicts into scheduler-level containment actions. The central design choice is
to treat agent security as a runtime verification problem over declared
behavioral boundaries rather than as a purely identity-based or
signature-matching problem.

Our evaluation shows that this approach is effective and practical. On the real
attestation path, \projectname~detects all four implemented attack classes with
sub-second median latency at the default 1\,s interval and low measured verifier
overhead. In the controlled studies, \projectname~achieves full coverage over the
four-attack suite, outperforms representative baselines, shows no observed false positives on the evaluated benign workflows, and shows clear
ablation evidence that the full system derives its coverage from multiple
complementary mechanisms rather than from a single rule.

The broader implication is that securing AI agents requires a shift from
verifying who the agent is to verifying what the agent is doing. For shared HPC
infrastructure, that shift is especially important because the attack surface is
large, the data is valuable, and legitimate workflows already require broad
access to files, tools, and external services. Behavioral attestation does not
eliminate prompt injection or tool compromise, but it does provide a concrete
way to detect and contain their effects before they become cluster-scale data
exfiltration events. We believe this is a necessary systems foundation for the
next generation of AI-enabled scientific computing.
